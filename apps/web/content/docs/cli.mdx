Install ilo from npm:

```sh
npm i -g iloso
```

Run `ilo --help` to see the current command tree.

## X account

```sh
ilo x connect
ilo x status
ilo x disconnect
```

`ilo start` is the guided shortcut for `ilo x connect`.

Before asking for a Client ID, the guided flow shows the X developer portal, recommends a Native App with OAuth 2.0 read and write access, and prints the exact default callback URL:

```txt
http://127.0.0.1:8976/callback
```

Copy the OAuth 2.0 Client ID from the app's **Keys and tokens** page. A Native App uses PKCE and does not need a client secret.

Your X client configuration and account metadata live in `~/.config/ilo/config.json`. OAuth tokens and the optional client secret are stored in the operating system keychain.

## Publish one post

```sh
ilo post --text "Post text"
ilo post --file ./post.txt
printf "Post from stdin" | ilo post
```

Interactive publishing asks for confirmation. Use `--yes` only when the caller has already confirmed the post.

## Reply and attach images

Pass an X post id or URL to publish a reply:

```sh
ilo post --reply-to https://x.com/example/status/123 --text "Reply text"
```

Attach up to four JPEG, PNG, or WebP files. Repeat `--image` and pair each file with `--alt` in the same order:

```sh
ilo post \
  --text "Weekly signups" \
  --image ./chart.png \
  --alt "Bar chart of weekly signups by source"
```

The same options work with `ilo drafts create`, including scheduled drafts. ilo saves absolute image paths with the draft, so the files must remain available until it publishes. Each image can be up to 5 MB.

X applies its own API access and conversation rules to replies. A reply that works in the X website can still be rejected by the API.

## Search public followers

Import public follower profiles through FxTwitter. The default command reads 20 pages and saves progress, so rerunning it resumes a large import:

```sh
ilo x followers sync adamwathan --pages 20
ilo x followers status adamwathan
```

Use `--all` to continue until FxTwitter reports the end of the list:

```sh
ilo x followers sync adamwathan --all
```

For a large account, run the same full import as a detached local process:

```sh
ilo x followers sync adamwathan --background
ilo x followers status adamwathan
```

The start command prints the progress log path. The status command shows the background PID while it is running. Every page is committed to local SQLite, so a stopped job can resume without starting over.

Read the complete public FxTwitter record stored for one imported follower:

```sh
ilo x followers profile adamwathan leerob
ilo x followers profile adamwathan leerob --json
```

The JSON result includes the public bio, location, website, join date, account counts, avatar, banner, verification details, and raw provider object. Older rows imported before these fields were added need a `--restart` sync to fill them in.

Search the local names, handles, bios, and locations with SQLite FTS5:

```sh
ilo x followers search adamwathan \
  --query "how many of my followers work at cursor|vercel|sentry"
```

Employer searches keep `current`, `former`, and `unclear` matches separate and return the public bio used as evidence. Check the coverage line first. A partial import only answers for the profiles indexed so far, and a public bio is not a verified employment record.

The normal result uses a terminal table. `--limit` controls how many evidence rows appear for each company without changing the totals:

```sh
ilo x followers search adamwathan \
  --query "works at cursor|vercel|sentry" \
  --limit 10
```

Export the matching public profiles when you want to inspect them in a spreadsheet or make a chart:

```sh
ilo x followers search adamwathan \
  --query "works at cursor|vercel|sentry" \
  --csv ./adam-company-followers.csv
```

The CSV has one row per company and profile match. It includes the classification, bio evidence, account counts, location, website, verification details, public image URLs, join date, and import coverage. Search classifies up to 10,000 candidates for each company by default. Change that cap with `--candidates`.

## Drafts

```sh
ilo drafts create --text "Draft text"
ilo drafts list
ilo drafts schedule <draft-id> --at "Friday at 2pm"
ilo drafts publish <draft-id>
```

Schedule values can be ISO timestamps or plain phrases such as `in 30 minutes`, `tomorrow at 9am`, and `Monday at 14:00`.

## Local scheduler

```sh
ilo scheduler run
ilo scheduler watch --interval 60
```

`run` checks once. `watch` keeps running and checks on an interval. Drafts stay in the local SQLite database until they are published or removed.

## JSON output

Commands intended for scripts support `--json`.

```sh
ilo x status --json
ilo drafts list --json
ilo scheduler run --json
```

## Local MCP

```sh
ilo mcp serve
```

This starts the local stdio MCP server for compatible agents.

## Agent skill

```sh
ilo skill install
```

The packaged ilo skill teaches coding agents when to use ilo, which CLI commands and MCP tools match the task, and which publishing actions require confirmation. Install it alongside the MCP server so an agent has both the tools and the instructions for using them safely.

## Safe automation

Publishing commands are interactive by default. A person sees the final text, reply destination, image paths, and alt text before ilo sends anything to X. Use `--yes` only in automation that already has an approval step. Draft creation, follower research, listing, and status checks do not publish anything.

For scripts, prefer JSON output and check the process exit code. Successful commands exit with code `0`; invalid input, missing credentials, and provider failures return a non-zero code with an actionable message.

```sh
draft_id=$(ilo drafts create --text "Review this first" --json | jq -r .id)
ilo drafts schedule "$draft_id" --at "2026-07-15T09:00:00+01:00"
```

## Scheduling behaviour

A scheduled draft remains local until a scheduler process finds it due. `ilo scheduler run` performs one check and exits, which suits cron and task runners. `ilo scheduler watch` stays open and is better for a laptop or always-on terminal session.

Natural-language times use your configured local timezone. Use an ISO timestamp with an offset when a post must go out at an exact instant, especially on a server or around daylight-saving changes.

## Files and credentials

ilo keeps configuration under `~/.config/ilo`, local working data in SQLite, and secrets in the system keychain. Back up the database if your draft history matters, but do not copy keychain exports into a repository or shell script.

If a command cannot find the connected account, run `ilo x status`. Reconnect with `ilo x connect` after changing X app scopes or rotating the client configuration.
