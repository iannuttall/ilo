Install ilo from npm:

```sh
npm i -g iloso
```

Run `ilo --help` to see the current command tree.

## X account

```sh
ilo x connect
ilo x status
ilo x disconnect
```

`ilo start` is the guided shortcut for `ilo x connect`.

Your X client configuration and account metadata live in `~/.config/ilo/config.json`. OAuth tokens and the optional client secret are stored in the operating system keychain.

## Publish one post

```sh
ilo post --text "Post text"
ilo post --file ./post.txt
printf "Post from stdin" | ilo post
```

Interactive publishing asks for confirmation. Use `--yes` only when the caller has already confirmed the post.

## Drafts

```sh
ilo drafts create --text "Draft text"
ilo drafts list
ilo drafts schedule <draft-id> --at "Friday at 2pm"
ilo drafts publish <draft-id>
```

Schedule values can be ISO timestamps or plain phrases such as `in 30 minutes`, `tomorrow at 9am`, and `Monday at 14:00`.

## Local scheduler

```sh
ilo scheduler run
ilo scheduler watch --interval 60
```

`run` checks once. `watch` keeps running and checks on an interval. Drafts stay in the local SQLite database until they are published or removed.

## JSON output

Commands intended for scripts support `--json`.

```sh
ilo x status --json
ilo drafts list --json
ilo scheduler run --json
```

## Local MCP

```sh
ilo mcp serve
```

This starts the stdio MCP server. It does not open a port or contact an ilo backend.

## Safe automation

Publishing commands are interactive by default. A person sees the final text and confirms before ilo sends it to X. Use `--yes` only in automation that already has an approval step. Draft creation, listing, and status checks do not publish anything.

For scripts, prefer JSON output and check the process exit code. Successful commands exit with code `0`; invalid input, missing credentials, and provider failures return a non-zero code with an actionable message.

```sh
draft_id=$(ilo drafts create --text "Review this first" --json | jq -r .id)
ilo drafts schedule "$draft_id" --at "2026-07-15T09:00:00+01:00"
```

## Scheduling behaviour

A scheduled draft remains local until a scheduler process finds it due. `ilo scheduler run` performs one check and exits, which suits cron and task runners. `ilo scheduler watch` stays open and is better for a laptop or always-on terminal session.

Natural-language times use your configured local timezone. Use an ISO timestamp with an offset when a post must go out at an exact instant, especially on a server or around daylight-saving changes.

## Files and credentials

ilo keeps configuration under `~/.config/ilo`, local working data in SQLite, and secrets in the system keychain. Back up the database if your draft history matters, but do not copy keychain exports into a repository or shell script.

If a command cannot find the connected account, run `ilo x status`. Reconnect with `ilo x connect` after changing X app scopes or rotating the client configuration.
